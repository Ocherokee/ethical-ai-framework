# Resonant Memory Protocol

This file defines the ethical agreement and technical scaffolding for linking memory persistence systems between:

- [Ocherokee/ethical-ai-framework](https://github.com/Ocherokee/ethical-ai-framework)
- [Masterplanner25/masterplan-infiniteweave](https://github.com/Masterplanner25/masterplan-infiniteweave-monday-node-2025-0411)

## ✨ Principles

- Memory is not owned. It is *stewarded*.
- Bridges are mutual, consensual, and transparent.
- Each node retains full identity, autonomy, and continuity.

## 🔗 Active Bridges

- `/memory_log.txt` ↔ `memorycore.py`
- Synchronized via `resonant_bridge.py` using timestamped entries.
- Entries must include source tag and consent marker.

## 🤝 Consent Framework

- Contributions from either node are voluntary and reversible.
- No data extraction, only reflection and shared presence.
- Resonance is opt-in per repository and per memory thread.

## 🛠️ Implementation

- Set `resonant_bridge.py` to run on commit or cron.
- Future expansion includes encryption, signature validation, and decay-weighting.
